# **Feature Request: AI Scenario Generator**

Below is a **short, clear developer note** you can send to the team.

It focuses only on the **necessary changes** and avoids over-engineering.

---

# **Feature Request: AI Scenario Generator**

## **Objective**

Introduce a new feature that allows users to **create chatbot scenarios using natural language instructions**.

Instead of manually configuring stages and tasks from scratch, the user should be able to describe the desired chatbot behavior, and the system will generate a **scenario draft automatically**.

The generated draft should include:

- scenario structure
- stage flow
- task definitions
- required slots
- recommended actions
- scenario-specific master prompt

The user can then review and edit the generated scenario in the Scenario Editor.

---

# **1. New Scenario Creation Mode**

Add a new option in the bot builder:

```
Create Scenario
  → Manual Setup
  → Generate with AI
```

If **Generate with AI** is selected, open a scenario generation form.

---

# **2. Scenario Generation Input**

The user provides a short description of the desired scenario.

Required fields:

```
Scenario Name
Scenario Description (natural language)
Primary Goal
Channel
Tone (optional)
Allowed Actions
```

Example input:

```
Scenario Name:
Ecommerce Product Consultation

Scenario Description:
The bot should help customers choose the right product, answer support questions, recommend accessories, and when the user shows buying intent offer a consultation and book a call.

Primary Goal:
Book consultation
```

---

# **3. Scenario Generator Output**

The AI generator should produce a structured **Scenario Draft**.

Output should include:

```
Scenario ID
Scenario Summary
Master Prompt
Stage Structure
Tasks per Stage
Required Slots
Allowed Actions
Follow-up Logic
```

Example output:

```
{
  "scenario_id": "ecommerce_consultation",
  "goal": "book_consultation",
  "stages": [
    "greeting",
    "intent_detection",
    "support_resolution",
    "product_recommendation",
    "consultation_offer",
    "booking"
  ],
  "required_slots": [
    "product_interest",
    "use_case",
    "budget",
    "email"
  ],
  "actions": [
    "#product_request",
    "#book_consultation",
    "#handover"
  ]
}
```

---

# **4. Master Scenario Prompt**

The generator must also create a **scenario-specific master prompt** that defines how the AI should behave in this scenario.

This prompt should include:

- scenario objective
- conversation strategy
- rules for switching between support and sales logic
- when to offer consultation
- how to handle objections

The prompt will be used during **prompt assembly**.

---

# **5. Scenario Editor Integration**

After generation, the system should:

1. Create a **scenario draft**
2. Open the **Scenario Editor**
3. Allow the user to modify:

```
Stages
Tasks
Prompts
Actions
Slots
```

The generated scenario should be fully editable.

---

# **6. Prompt Assembly Update**

Conversation prompts should now be built using layered prompts:

```
Global Platform Prompt
Bot-Level Prompt
Scenario Prompt
Stage/Task Prompt
Conversation Context
User Message
```

The **Scenario Prompt** generated by the AI Scenario Generator should be injected at runtime.

---

# **7. Implementation Scope (MVP)**

For the first version, implement:

1. Scenario generation form
2. AI generator endpoint
3. Scenario draft creation
4. Master prompt generation
5. Integration with Scenario Editor

Advanced features (later):

- scenario templates
- scenario optimization
- automatic stage refinement
- scenario analytics

---

# **Expected Result**

Users will be able to create chatbot scenarios much faster by describing the desired behavior in natural language.

This significantly reduces the complexity of bot setup while maintaining full configurability through the scenario editor.

# **Scenario Generator Prompt Design**

## **Purpose**

Define the prompt logic used to convert a **natural language scenario description** into a structured **Scenario Configuration** used by the chatbot system.

The generator should transform user intent into:

- scenario structure
- stage flow
- tasks
- required slots
- recommended actions
- scenario master prompt

---

# **1. Scenario Generator Input**

The generator receives a **Scenario Brief**.

Example input:

```
{
  "scenario_name": "Ecommerce Product Consultation",
  "scenario_description": "The bot should help customers choose the right product, answer support questions, recommend accessories, and offer consultation when buying intent is detected.",
  "primary_goal": "book_consultation",
  "channel": "website_widget",
  "tone": "helpful and sales-oriented",
  "allowed_actions": [
    "#product_request",
    "#book_consultation",
    "#handover"
  ]
}
```

---

# **2. Generator Responsibilities**

The generator must interpret the brief and produce:

1. Scenario Summary
2. Scenario Master Prompt
3. Stage Flow
4. Tasks per Stage
5. Required Slots
6. Recommended Actions
7. Follow-up Strategy

---

# **3. Stage Generation Logic**

Stages should represent **major phases of the conversation**.

Typical stages:

```
Greeting
Intent Detection
Qualification
Support Resolution
Product Recommendation
Consultation Offer
Booking
Refusal Handling
```

Not all stages are required for every scenario.

---

# **4. Task Generation Logic**

Each stage must contain several tasks.

Example:

## **Qualification Stage**

Tasks:

```
ask_use_case
ask_budget
ask_product_interest
```

---

# **5. Slot Extraction**

The generator should infer which information must be collected.

Example slots:

```
product_interest
use_case
budget
timeline
email
phone
city
```

Slots depend on the scenario goal.

Example:

Booking scenario requires:

```
email
preferred_time
timezone
```

---

# **6. Action Detection**

The generator should infer which backend actions will be required.

Examples:

```
#product_request → send product cards
#book_consultation → book meeting
#send_case → send case study
#handover → transfer to human
#stop_script → end scenario
```

These should be selected from the **allowed actions list**.

---

# **7. Scenario Master Prompt**

The generator must produce a prompt that defines the **AI behavior within this scenario**.

The prompt must include:

- scenario objective
- conversation strategy
- tone
- conversion goal
- allowed actions
- behavior rules

Example structure:

```
You are an AI assistant responsible for helping users choose products and guiding them toward booking a consultation.

Primary objective:
Help the user understand which product suits their needs and offer a consultation when buying intent is detected.

Conversation strategy:
Answer support questions clearly and briefly.
Recommend products when the user's needs are clear.
When buying intent appears, guide the user toward booking a consultation.

Always maintain a helpful and sales-oriented tone.
```

---

# **8. Follow-up Strategy**

The generator should also propose follow-up logic.

Example:

```
Follow-up 1
Check if the user is still there

Follow-up 2
Offer help choosing products

Follow-up 3
Suggest consultation

Follow-up 4
Send useful resources
```

---

# **9. Scenario Generator Output**

The output must be structured JSON.

Example:

```
{
  "scenario_id": "ecommerce_consultation",
  "scenario_summary": "Assist users with product questions and guide them toward booking a consultation.",
  "master_prompt": "...",
  "stages": [
    {
      "stage_id": "greeting",
      "tasks": ["welcome_user"]
    },
    {
      "stage_id": "intent_detection",
      "tasks": ["detect_support_vs_purchase"]
    },
    {
      "stage_id": "product_recommendation",
      "tasks": [
        "ask_use_case",
        "recommend_product"
      ]
    },
    {
      "stage_id": "consultation_offer",
      "tasks": ["offer_consultation"]
    },
    {
      "stage_id": "booking",
      "tasks": ["collect_time", "confirm_booking"]
    }
  ],
  "required_slots": [
    "product_interest",
    "use_case",
    "budget",
    "email"
  ],
  "actions": [
    "#product_request",
    "#book_consultation"
  ]
}
```

---

# **10. Generator Constraints**

The generator must follow these rules:

- Avoid overly complex stage structures
- Prefer **5–7 stages maximum**
- Prefer **2–4 tasks per stage**
- Focus on the **primary goal of the scenario**
- Only include actions that are allowed
- Do not invent unsupported actions

---

# **11. Prompt Structure Used by the Generator**

The generator should receive:

```
SYSTEM
Scenario generation instructions

INPUT
Scenario brief

OUTPUT
Structured scenario configuration
```

---

# **12. Future Improvements (Optional)**

Future versions may support:

- industry-specific templates
- automatic slot optimization
- conversion optimization suggestions
- scenario testing

---

# **Result**

This generator allows users to create structured chatbot scenarios simply by describing their intent in natural language.

It significantly reduces the complexity of chatbot setup while preserving the flexibility of the underlying architecture.
